# CS2 T+7 Screening Skill

## Use When

用户要求分析 CS2 饰品、T+7、SteamDT、CSQAQ、买什么、每日扫描、持仓复盘时使用。

## Required Inputs

- 用户的价格上限、板块偏好、持仓情况。
- `.env` 中的 `STEAMDT_API_KEY`。
- 观察池：优先读取 `config/watchlist.csv`，没有则使用用户临时给出的候选品。
- 规则文档：`docs/cs2_t7_rules.md` 和 `docs/risk_management.md`。

## Workflow

1. 确认扫描范围：默认使用 discovery；只有用户明确要求时才只扫观察池、指定板块或持仓。
2. 获取数据：SteamDT 单品价格、批量价格、均价、K 线；CSQAQ 大盘和板块指数。
3. 计算指标：7 日涨幅、30 日涨幅、30 日高点回撤、30 日低点反弹、点差、深度、板块强弱。
4. 按 T+7 评分排序，不只看短期涨幅。
5. 分成爆发票、稳健票、观察票、不碰票。
6. 给出买入平台、买入价区间、减仓线、目标价、仓位建议、T+7 到期处理。
7. 保存快照和报告。

## Universe Rules

`config/watchlist.csv` 是手工种子池，只能代表观察池，不能代表全市场。由于市场变化快，默认不要把它当作推荐来源主体。

默认分析不要碰箱子、收藏包、胶囊，除非用户单独点名。

默认只考虑：

- 崭新出厂 / Factory New 的皮肤。
- 全息 / Holo 的单张印花。

默认排除：

- 非崭新出厂磨损。
- 非全息印花。
- 无法确认磨损的非印花标的。

如果只扫观察池，输出时必须说“观察池内排名”，不能说“全市场最优”。

更完整的流程是：

1. 每次扫描都先用 CSQAQ 列表、排行榜或全量价格数据做 discovery 粗筛。
2. 排除箱子、收藏包、胶囊。
3. 把候选交给 SteamDT 做平台价、求购、卖单和 K 线验证。
4. 只有通过流动性和 T+7 风险检查的标的才加入观察池。

## Output Rules

每个推荐必须说明：

- 数据来源和扫描时间。
- 推荐类别。
- 买入理由。
- 主要风险。
- 无效条件。
- T+7 解锁后的处理规则。

如果接口配额受限，必须说明扫描不是全市场最优。
