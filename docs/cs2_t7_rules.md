# CS2 饰品 T+7 分析规则

## 数据源

优先使用当前运行中的实时数据：

- SteamDT：单品价格、批量价格、均价、K 线、基础信息。
- CSQAQ：饰品总指数、贴纸、主战、红皮、赛事贴纸等板块指数。
- 本地缓存：仅在实时数据不可用时作为参考，报告中必须标记为缓存。

SteamDT API key 必须来自 `.env` 或本地安全配置，不允许写死在代码里。

## 默认排除范围

除非用户单独点名要求，否则扫描器和人工分析默认不碰：

- 箱子 / 武器箱：`case`、`weapon case`。
- 收藏包 / 纪念包：`collection package`、`souvenir package`。
- 胶囊：`capsule`。

原因：

- 这几类标的更容易受活动、掉落、开箱量、库存预期和短期情绪影响。
- T+7 锁定期内，如果题材热度衰退，退出风险可能比普通皮肤更高。
- 后续如果要单独做箱子/胶囊系统，应该用另一套规则：开箱量、掉落池、赛事周期、存量变化和活动日历。

默认优先分析：

- 崭新出厂 / Factory New 的主战皮肤。
- 崭新出厂 / Factory New 的手枪和常用副武器皮肤。
- 崭新出厂 / Factory New 的红皮/粉皮等高流动性皮肤。
- 单张全息 / Holo 印花可以观察，但不把普通印花或胶囊当作默认候选。

默认不分析：

- 非崭新出厂磨损的皮肤。
- 非全息印花。
- 无法确认磨损的非印花标的。

## 候选池来源约束

`config/watchlist.csv` 只是手工种子池，不代表全市场。

由于 CS2 饰品市场变化快，默认推荐流程应优先使用联合 `discovery`：

```text
CSQAQ + SteamDT base + 观察池 + 最近快照 -> 排除箱子/收藏包/胶囊 -> 只保留崭新出厂皮肤和全息印花 -> SteamDT 深度验证 -> T+7 排序
```

默认报告必须标明扫描范围：

- 观察池扫描：只代表当前观察池内的排序。
- 发现扫描：代表从 CSQAQ 列表/排行榜粗筛后得到的候选池。
- 全市场扫描：只有在 API 权限和配额足够、并且完成全市场枚举时才能使用这个说法。

如果只扫观察池，不允许把结论描述成“全市场最优”。

观察池只能作为复查列表和持仓跟踪列表，不能限制新机会发现。

## SteamDT 字段

项目默认使用以下信息：

- `marketHashName`：跨价格和 K 线接口使用的物品名。
- 平台最低价：当前可买成本参考。
- 求购价：退出流动性参考。
- 卖单数量：供应压力参考。
- 7 日和 30 日 K 线：趋势、回撤、反弹计算。
- K 线结构分析：由 `scripts/analyze_cs2_kline.py` 输出趋势、结构状态、支撑阻力、入场区、图表止损和确认条件。

已知接口形状：

- Host: `https://open.steamdt.com`
- Auth: `Authorization: Bearer <API_KEY>`
- 常用接口：`/open/cs2/v1/price/single`、`/open/cs2/v1/price/batch`、`/open/cs2/v1/price/avg`、`/open/cs2/item/v1/kline`、`/open/cs2/v1/base`
- 批量价格：`marketHashNames` 使用 JSON 数组。
- SteamDT base：`/open/cs2/v1/base` 用作 CSQAQ 异常时的候选宇宙补充，并写入 24 小时本地缓存。

## 核心指标

每个候选品至少计算：

- 7 日涨幅：短期爆发强度。
- 30 日涨幅：中期趋势。
- 30 日高点回撤：追高风险。
- 从 30 日低点反弹：是否已经过度拉升。
- 当前价与 7 日均价偏离：短期是否过热。
- 当前价与 30 日均价偏离：中期是否偏离过大。
- 点差：最低卖价与最高求购价之间的差距。
- 卖单深度：上方供应是否压制。
- 求购深度：T+7 后能否顺利退出。
- 板块强弱：单品是否顺应大盘和板块。

## T+7 评分

当前脚本评分近似 100 分，并允许图表结构做小幅调整：

- 动量 40 分：7 日涨幅、30 日涨幅、从 30 日低点反弹。
- 抗风险 25 分：30 日高点回撤、7/30 日均价偏离、T+7 锁定风险。
- 流动性 20 分：点差、求购深度、卖单深度。
- 板块 10 分：CSQAQ 总指数和所属板块强弱。
- 数据质量 5 分：实时数据完整度、K 线覆盖、是否使用缓存。
- 图表结构调整：通常在 -4 到 +4 附近，来自 `chart_analysis.score_adjustment`。

默认排序不是“涨得最多”，而是：

```text
综合排序 = 上涨爆发 + 抗风险能力 + T+7 可退出性 + 图表结构确认
```

图表结构不能覆盖硬约束：点差过大、买盘深度不足、同平台退出不清楚、价格超过上限时，不能因为图表形态好就升级为买入。

## 分组标准

爆发票：

- 7 日涨幅强于板块。
- 30 日趋势向上。
- 当前回撤不深，或回踩后重新放量。
- 点差和卖单深度仍可接受。
- 风险是追高，必须给更紧的减仓线。

稳健票：

- 7 日涨幅中等但趋势稳定。
- 30 日回撤浅。
- 求购深度好，点差小。
- 板块不弱。
- 适合标准仓或核心仓。

观察票：

- 有板块或形态线索，但入场价格不理想。
- K 线不足、指数缺失或流动性需要确认。
- 等待回踩、突破确认或点差收窄。

不碰票：

- 7 日涨幅过热且远离均价。
- 30 日高点回撤大，反弹弱。
- 点差过大，求购深度不足。
- 卖单堆积明显。
- 所属板块弱于大盘。

## 输出模板

每日报告必须包含：

- 扫描时间和数据来源。
- 市场概况：CSQAQ 总指数和重点板块。
- 推荐列表：爆发票、稳健票、观察票、不碰票。
- 每个推荐品：买入平台、买入价区间、止损/减仓线、目标价、仓位建议、T+7 到期处理。
- 图表结构：趋势、结构状态、支撑阻力、图表入场区、图表止损/目标、确认条件。
- 风险说明：数据缺口、配额限制、异常波动。

## 配额受限处理

如果 `/open/cs2/v1/base` 或其他接口配额受限：

- 改用最近快照、`config/watchlist.csv` 或高流动性候选池。
- 报告必须写明“非全市场扫描”。
- 不允许把结果描述成全市场最优。
