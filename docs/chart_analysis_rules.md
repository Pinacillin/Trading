# 图表截图分析规则

## 目标

用 TradingView / Thinkorswim / MT4/MT5 图表截图或 OHLCV 数据，按固定流程输出交易计划。

Codex 必须区分：

- 观察：图上已经发生了什么。
- 判断：趋势、结构、支撑阻力。
- 计划：入场、止损、止盈、仓位。
- 确认：还需要等待什么信号。
- 无效：什么情况说明分析失效。

## 输入要求

截图最好包含：

- 标的名称。
- 时间周期。
- 当前价格。
- K 线区域。
- 成交量或常用指标。
- 用户已经画好的支撑阻力或趋势线。

如果截图缺少关键信息，Codex 要明确写出缺失项，不允许假装知道。

## 分析流程

1. 识别市场和周期：标的、时间框架、是否为盘中或收盘图。
2. 判断趋势：上涨、下跌、震荡。
3. 标出结构：前高、前低、HH/HL、LH/LL、突破、回踩。
4. 标出关键价位：支撑、阻力、流动性区域、失效位。
5. 识别形态：趋势延续、突破、假突破、箱体、回调、反转。
6. 计算交易计划：入场、止损、止盈、风险回报比。
7. 给出类别：做多计划、做空计划、等待计划、不交易。
8. 记录复盘字段：截图路径、分析时间、计划和无效条件。

## 趋势规则

上涨趋势：

- 价格形成更高高点和更高低点。
- 当前价位在关键均线或结构支撑上方。
- 回踩不破前低，反弹有动能。

下跌趋势：

- 价格形成更低高点和更低低点。
- 当前价位在关键均线或结构阻力下方。
- 反弹不过前高，回落有动能。

震荡：

- 高低点无序。
- 价格在箱体中部。
- 上下边界不清晰或假突破频繁。

## 支撑阻力规则

优先级：

1. 明显前高前低。
2. 多次反应的水平区间。
3. 放量突破或跌破的位置。
4. 缺口、成交密集区、整数关口。
5. 用户图上已经标出的区域。

支撑阻力应该标成区间，不要只给一个孤立价格。

## 交易计划规则

做多计划必须包含：

- 入场：突破确认或回踩确认。
- 止损：结构低点下方或 ATR 之外。
- 止盈：前高、阻力区、固定 R 倍或分批目标。
- 无效：跌破哪一处说明多头失败。

做空计划必须包含：

- 入场：跌破确认或反弹受阻。
- 止损：结构高点上方或 ATR 之外。
- 止盈：前低、支撑区、固定 R 倍或分批目标。
- 无效：突破哪一处说明空头失败。

等待计划必须说明：

- 为什么现在不交易。
- 等待什么价格或形态。
- 触发后如何重新评估。

## 输出模板

```text

关键支撑：
关键阻力：

止损 / 无效位：
止盈目标：
风险回报比：
仓位建议：
确认信号：
不交易理由：
复盘记录：
```
